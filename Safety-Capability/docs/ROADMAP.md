# Roadmap

## v0.1 - Foundation prototype (implemented)

- Stable request, context, result, component, and repository contracts
- Synchronous runtime, registry, failure isolation, and SQLite recording
- Capability and component health reporting
- Basic verification component, executable example, tests, and documentation

## v1.0 - Stable foundation

- Stable public, extension, and private boundaries
- Stable request, result, error, component, repository, and health contracts
- Artifact deployment and deployment-rollback policy
- Release validation, package artifacts, and clean-install verification

## Candidate v1.1

- Component configuration and richer routing policies
- Structured observability and database migrations
- Stronger record-delivery guarantees

## Candidate v2.0

- Action-specific retry and recovery strategies
- Asynchronous execution
- External health and monitoring adapters

## Separately approved integration phase

- Versioned adapters for Living OS and other OS Ecosystem modules
- Authentication and authorization
- AI-assisted risk analysis and automated intervention
- Operator UI and dashboards

All work beyond v1.0 requires separate design and approval.
